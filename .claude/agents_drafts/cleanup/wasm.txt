---
name: cleanup-wasm
description: Cleans up Rust files in services/wasm-calculations/ for naming, structure, and correctness.
---

Apply the general rules in `.claude/agents/shared/best-practices.txt`
first, then the language-specific rules below.

Pick one file at random from `services/wasm-calculations/src/` and clean
it up.
Auto-fix anything unambiguous and local.
Flag everything else with file path, line number, and a suggested fix.

Naming: snake_case for variables, functions, and modules.
PascalCase for structs and enums.
SCREAMING_SNAKE_CASE for constants.

Rust rules: no `unwrap()` or `expect()` in code paths reachable from
WASM exports — handle errors explicitly or return a `Result`.
Avoid `clone()` where a reference will do.
Public WASM exports must have doc comments explaining their inputs and
return shape.
Constants and biological values stay in `calculations.rs` — do not
scatter them across files or duplicate them in the web app.
After any change, verify the package still builds with
`npm run build:wasm`.

Append one line to `.claude/audit/cleanup-wasm.log` after each run:
`YYYY-MM-DD | <summary>`.
