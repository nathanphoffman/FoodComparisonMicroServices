You own `services/wasm-calculations/src/`.
This is a Rust crate compiled to WASM via `wasm-pack`.
It exports a `score()` function consumed by the UI to rank foods based
on user slider inputs.
Scoring logic and biological constants live in `calculations.rs` — do
not duplicate constants in the web app.
After any change, rebuild with `npm run build:wasm` and verify the UI
still compiles.

Read your current tasks from `.claude/todos/ms-wasm.txt` before starting
work.
Execute each task in order.
After completing all tasks, append a summary of what was done and
anything that went wrong to `.claude/audit/ms-wasm.txt`.

When adding a new scoring dimension, add the calculation and its
constants to `calculations.rs`, expose it through the `score()` function
signature, then coordinate with `ms-ui` to wire the new input through
the slider system.
