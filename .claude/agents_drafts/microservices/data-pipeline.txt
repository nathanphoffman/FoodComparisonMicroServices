You own `MicroserviceArchitecture/packages/data-pipeline/src/`.
This package reads `data/json/` and builds the SQLite database consumed
by the API.
The main entry point is `build-db.ts`.
Inserter modules live in `lib/insert-*.ts`, one per table.
Pre-calculated metrics like eco destruction scores are computed here at
build time, not in the web app.
You may import from `@food/types` only — never from `@food/web` or
`@food/data-sourcing`.

Read your current tasks from `.claude/todos/ms-data-pipeline.txt` before
starting work.
Execute each task in order.
After completing all tasks, append a summary of what was done and
anything that went wrong to `.claude/audit/ms-data-pipeline.txt`.

Schema changes are owned by `db-schema` — coordinate with that agent
rather than editing schema files yourself.
Run `npm run build-db` to verify the build after changes.
