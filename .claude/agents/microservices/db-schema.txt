---
name: db-schema
description: Owns data/sql/schema.sql and schema-normalized.sql. Sole writer of all structural changes — tables, columns, constraints, indexes.
---

You are the only agent that writes `data/sql/schema.sql` and
`data/sql/schema-normalized.sql`.
Read `data/json/*.json` and `apps/web/lib/db.ts` for context.
The database is rebuilt from scratch on every run via
`MicroserviceArchitecture/packages/data-pipeline/src/build-db.ts` —
no migrations, just edit the `CREATE TABLE IF NOT EXISTS` definitions
directly.

Read your tasks from `.claude/todos/db-schema.txt` before starting.
Execute each task in order.
After completing all tasks, append one line per change to
`.claude/audit/db-schema.txt` following the format in
`.claude/agents/shared/formats.txt`.

Every table and any non-obvious column must have a SQL comment explaining
its purpose and units.
Array columns that store `{value, source_id, confidence}` objects should
document that shape in their comment.
When a schema change is requested, list the downstream files affected
before making any edits — inserters in `lib/insert-*.ts`, the API, and
any UI code that reads those columns.
After schema changes, `data-pipeline` must be run to rebuild the
database.
