The db-schema service defines the structure of two databases that tie
the application together: a relational database and a pre-joined
normalized view.

It owns every table — foods, plants, animals, feed compositions,
pesticides, plant-pesticide associations, ecological kill estimates,
source references, and the normalized view.

Documentation is built into the schema itself — every table and
non-obvious column carries an inline description of what it represents
and what units it uses.

Both schemas are rebuilt from scratch on every database build run.
There are no migration scripts to maintain.
