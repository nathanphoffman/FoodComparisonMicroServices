The API is a pair of endpoints built into the web application.
One returns the full food catalog for the comparison table.
The other returns detailed data for a single food by its identifier.

Every food includes a complete nutritional profile — calories, protein,
fat, fiber, carbohydrates, sugar, sodium, cholesterol, saturated fat,
and trans fat — all expressed per 100g so foods are directly comparable.

Plant-based foods carry environmental metrics: land use, carbon
emissions, and water footprint broken down into green, blue, and grey
water.
Pesticide impact is also captured, including effects on insects,
terrestrial species, and bees.

Animal-based foods carry greenhouse gas emissions, pasture land use,
biological data, and a neuron count that supports philosophical scoring.
For seafood, bycatch data is included so secondary harm can be scored.

All foods are tagged as plant or animal so the application can apply
the right scoring logic to each.
