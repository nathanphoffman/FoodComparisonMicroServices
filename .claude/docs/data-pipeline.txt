The data pipeline builds the database that powers the rest of the
application.
It reads structured food data files and produces a fully populated
database that the API serves to the web application.

The pipeline covers every data category the application needs: base
food records, plant and animal metrics, feed compositions, pesticide
registries, plant-pesticide associations, ecological kill estimates,
source references, and a pre-joined normalized view.

The database is rebuilt in full every time the pipeline runs.
There are no partial updates — the entire dataset is loaded from
scratch, so the database always reflects the current state of the
source files.

Validation runs during import, so errors in the source files are
caught before they reach the database.
